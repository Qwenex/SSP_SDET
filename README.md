## Задание U-1. Тест-кейсы
1.	Получить от ментора один или более тест-кейсов для автоматизации.
2.	Создать собственный проект автотестов для way2automation с правильной структурой,\
использованием TestNG, паттернов (POM, Factory).

* Тест-кейсы:

### 1. [Главная страница](https://www.way2automation.com/)

#### 1.1 Проверка открытия страницы
- Все основные элементы отображаются:
    - Хедер с контактной информацией
    - Блок с навигацией
    - Кнопка регистрации
    - Список курсов (под текстом "Best Selenium Certification Course Online")
    - Футер

#### 1.2 Проверка хедера с контактной информацией
- Хедер содержит:
    - Номера телефонов
    - Ссылку на Skype
    - Адрес почты
    - Ссылки на социальные сети

#### 1.3 Блок с курсами (Most Popular Software Testing Courses)
- Проверка кнопок навигации (вперед и назад):
    - Работают корректно (меняют слайды)

#### 1.4 Футер
- Футер отображается и содержит:
    - Адрес
    - Номера телефонов
    - Email-адреса

### 2. Меню навигации при скроллинге
- Отображение меню при скроллинге страницы вниз:
    - Меню должно оставаться видимым после прокрутки страницы.

### 3. Проверка перехода по меню навигации на другие страницы
**Пример:** перейти на страницу All courses → Lifetime membership.
- Проверить, что произошёл переход на [новую вкладку](https://www.way2automation.com/lifetime-membership-club/)
- В заголовке содержится текст: **LIFETIME MEMBERSHIP CLUB**

### 4. [Страница авторизации:](https://www.way2automation.com/angularjs-protractor/registeration/#/login)
#### 4.1 Проверка полей ввода
- Поля "Username" и "Password" отображаются
- Кнопка Login задизейблена при незаполненных полях

#### 4.2 Проверка успешной авторизации
- Ввести валидные username и password
- Нажать на кнопку Login
- **Ожидаемый результат:** отображается сообщение об успешной авторизации ("You're logged in!!")

#### 4.3 Проверка авторизации с невалидными данными
- Ввести невалидный username или password
- Нажать Login
- **Ожидаемый результат:** отображается сообщение об ошибке ("Username or password is incorrect")

#### 4.4 Проверка успешного разлогирования
- Нажать на кнопку Logout
- **Ожидаемый результат:** отображаются поля для входа

### 5. [Банковское приложение:](https://www.way2automation.com/angularjs-protractor/banking/#/login)
#### 5.1 Перейти в интерфейс Sample Form
- Ввести корректные данные для регистрации
- В пункте Hobbies выбрать Sports
- В пункте About Yourself ввести текст:  
  "Самое длинное слово из предложенных хобби - N", где N — программно вычисленное самое длинное слово из пункта Hobbies
- Нажать кнопку Register
- Проверить появление сообщения: "User registered successfully!"

#### 5.2 Перейти в интерфейс Bank Manager Login
#### 5.2.1 Добавление покупателя
- Нажать Add Customer
- Заполнить корректными данными
- Нажать кнопку "Add Customer"
- Проверить появление всплывающего окна с подтверждением

#### 5.2.2 Открытие аккаунта
- В поле Customer выбрать созданный аккаунт покупателя из прошлого пункта
- Выбрать любую валюту
- Нажать кнопку "Process"
- Проверить появление всплывающего окна с подтверждением

#### 5.3 Перейти в интерфейс Customer Login
- Выбрать созданного покупателя
- Нажать "Login"
- Убедиться, что вошли в нужный аккаунт и на экране есть приветствие:  
  "Welcome *Имя вашего клиента* !!"

#### 5.3.1 Успешное пополнение счета
- Нажать кнопку "Deposit"
- Ввести сумму 100321
- Нажать на кнопку подтверждения "Deposit"
- Проверить появление сообщения "Deposit Successful"
- Перейти в "Transactions"
- Проверить наличие пополнения на 100321

#### 5.3.2 Неуспешное пополнение счета
- Нажать кнопку "Deposit"
- Ввести сумму 0
- Нажать на кнопку подтверждения "Deposit"
- Проверить отсутствие сообщения "Deposit Successful"
- Перейти в "Transactions"
- Проверить отсутствие пополнения на 0

#### 5.3.3 Успешное снятие средств
- Получить состояние баланса счета "Balance : ..."
- Нажать кнопку "Withdrawn"
- Ввести рандомную сумму N (от 1 до максимума на балансе)
- Нажать на кнопку подтверждения "Withdrawn"
- Проверить наличие сообщения "Transaction successful"
- Перейти в "Transactions"
- Проверить наличие снятия средств на сумму N

#### 5.3.4 Неуспешное снятие средств
- Нажать кнопку "Withdrawn"
- Ввести сумму 1000000
- Нажать на кнопку подтверждения "Withdrawn"
- Проверить наличие сообщения:  
  "Transaction Failed. You can not withdraw amount more than the balance."
- Перейти в "Transactions"
- Проверить отсутствие снятия средств на 1000000

#### 5.3.5 Проверка баланса
- Получить состояние баланса счета "Balance : ..."
- Нажать кнопку "Transactions"
- Сделать подсчет баланса из таблицы
- Проверить, что данные сходятся

#### 5.3.6 Снятие оставшихся средств
- Получить состояние баланса счета "Balance : ..."
- Нажать кнопку "Withdrawn"
- Ввести сумму, оставшуюся на счету
- Нажать на кнопку подтверждения "Withdrawn"
- Проверить наличие сообщения "Transaction successful"
- Проверить, что отображается "Balance : 0"

#### 5.3.7 Очистка истории транзакций
- Нажать кнопку "Transactions"
- Получить количество транзакций
- Нажать кнопку "Reset"
- Проверить, что транзакции очистились
- Нажать кнопку "Back"
- Проверить состояние баланса "Balance : 0"

### 5.4 Удаление покупателя
- Перейти в интерфейс Bank Manager Login
- Нажать кнопку "Customers"
- В поле поиска ввести First Name клиента из прошлых пунктов
- Проверить, что он нашелся
- Нажать кнопку "Delete"
- Очистить поле поиска
- Проверить, что в таблице он отсутствует
