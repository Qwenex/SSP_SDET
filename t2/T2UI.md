### Bug Report №1
* **Заголовок:**\
Грамматические ошибки в сообщении о пустом результате поиска: \
неверный предлог и регистр слова "Диске"

* **Описание:**\
  Обнаружены грамматические ошибки в системном сообщении при пустом результате поиска:
  1. Неверный предлог: Используется "На" вместо корректного "В"
  2. Неверный регистр: Слово "диске" написано с маленькой буквы вместо заглавной
  
  * Текущее сообщение: \
    "На вашем диске не найдено файлов или папок по запросу <'пример'>"
  * Ожидаемое сообщение: \
    "В вашем Диске не найдено файлов или папок по запросу <'пример'>"

* **Среда:**
    * Приложение: Яндекс Диск (Веб-версия)\
      Версия: Актуальная версия на 21.11.2025
    * Браузер: Yandex Browser \
      Версия: 25.10.1.1136 (64-bit)
    * OC: Windows 10
    * Разрешение экрана: 1920х1080

* **Шаги для воспроизведения:**
  1. Открыть главную страницу Яндекс Диска.
  2. В поле поиска ввести заведомо несуществующий запрос (например, object_123)
  3. Нажать Enter или кликнуть на значок лупы для выполнения поиска

* **Ожидаемый результат:**\
  "В вашем Диске не найдено файлов или папок по запросу <'object_123'>"
* **Фактический результат:**\
    "На вашем диске не найдено файлов или папок по запросу <'object_123'>"

* **Серьезность:**\
Trivial
* **Приоритет:**\
Low
  * **Приложения:**\
          1. Скриншот: ![Empty_Search_Massage.png](..%2Fscreenshots%2Ft2%2FbugReportsUi%2FscrBugReport1.png)
            - Сообщение о пустом результате поиска

### Bug Report №2
* **Заголовок:**\
Поиск не находит файлы в Корзине при работе из раздела "Файлы" (не соответствует требованием задания)

* **Описание:**\
Согласно требованиям задания, система поиска должна находить все файлы и папки, \
включая находящиеся в Корзине, независимо от текущего раздела.\
При поиске из раздела "Файлы" объекты, находящиеся в Корзине, не отображаются в результатах. \
Объекты находятся только при непосредственном поиске внутри раздела "Корзина".

* **Среда:**
    * Приложение: Яндекс Диск (Веб-версия)\
      Версия: Актуальная версия на 21.11.2025
    * Браузер: Yandex Browser \
      Версия: 25.10.1.1136 (64-bit)
    * OC: Windows 10
    * Разрешение экрана: 1920х1080

* **Шаги для воспроизведения:**
  1. Открыть главную страницу Яндекс Диска\
  2. Создать новый файл с именем `test_search_123.docx` в основном разделе Яндекс Диска
  3. Удалить файл `test_search_123.docx` (переместив его в Корзину)
  4. Ввести в поисковую строку `test_search_123.docx`
  5. Нажать Enter или кликните на значок лупы для выполнения поиска

* **Ожидаемый результат:**\
В результатах поиска отображается файл `test_search_123` с пометкой, что он находится в Корзине
* **Фактический результат:**\
Поиск возвращает пустой результат, файл `test_search_123` не отображается
 
* **Серьезность:**\
  Minor
* **Приоритет:**\
  Low
* **Приложения:**
  * Скриншоты: \
    1. ![Basket_With_File](..%2Fscreenshots%2Ft2%2FbugReportsUi%2FscrBugReport2.png)
       -Корзина с удаленным файлом \
    2. ![Empty_Search_Deleted_File](..%2Fscreenshots%2Ft2%2FbugReportsUi%2FscrBugReport3.png) 
       -Пустой поиск из раздела файлы

* **Примечания тестировщика:**\
Скорее всего такой функционал и задумывался изначально, \
и это просто некорректные требования к тестированию этого функционала
