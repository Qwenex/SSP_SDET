## Верхне-уровневое описание тест-кейсов согласно тест-плану Т4
Тестирование микро-сервисной системы.

### Группа 1: Базовые операции взаимодействия сервисов
1. **Успешное подключение услуги** \
   Операция подключения услуги с полным циклом от REST-запроса к сервису A, \
   до подтверждения от сервиса B с обновлением статуса
2. **Успешное отключение услуги** \
   Операция отключения услуги с обновлением статуса после ответа от сервиса B
3. **Успешное блокирование услуги** \
   Операция блокировки услуги с обновлением статуса после ответа от сервиса B

### Группа 2: Управление состоянием услуг
1. **Статус после успеха** \
   Статус услуги обновляется только после подтверждения от сервиса B
2. **Независимость запросов сервиса А** \
   Сервис A принимает запросы независимо от состояния обработки сервисом В
3. **Переключение статуса услуги** \
   Статус услуги должен переключаться с любого изначального статуса из 3 возможных на 2 оставшихся 

### Группа 3: Упорядочивание команд
1. **Последовательность команд** \
   Несколько команд выполняются в порядке их отправки
2. **Блокировка новой команды** \
   Новая команда не отправляется пока предыдущая не обработана
3. **Приоритет операций** \
   Проверка порядка при смешанных типах команд. \
   Команды выполняются в соответствии заданным им приоритетам

### Группа 4: Интеграция с RabbitMQ
1. **Доставка сообщений** \
   Сообщения доставляются от сервиса A к сервису B и обратно через RabbitMQ
2. **Формат AMQP-сообщений** \
   Корректность структуры и данных в AMQP-сообщениях
3. **Очередь команд** \
   Сообщения не теряются и сохраняют порядок в очереди

### Группа 5: Обработка ошибок
1. **Получение кода ошибки при недоступности сервиса А** \
   Получение кода ошибки при недоступности REST API
2. **Невалидные REST-запросы** \
   Получение кода ошибки при невалидных значениях в REST запросе для сервиса А
3. **Получение кода ошибки при недоступности сервиса B** \
   Поведение системы при недоступности сервиса B
4. **Потеря связи с RabbitMQ** \
   Поведение сервисов А и В при недоступности брокера сообщений
5. **Таймауты операций** \
   Поведение сервисов при очень долгом выполнении какой-либо операции (Таймауты)
